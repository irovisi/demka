## Модуль 1. Различия

### VLAN ID

| VLAN | В1 | В2 | В3 | В4 |
|---|---|---|---|---|
| HQ-SRV | 111 | 112 | 113 | 114 |
| HQ-CLI | 211 | 212 | 213 | 214 |
| Управление | 811 | 812 | 813 | 814 |

Названия сабинтерфейсов строятся по той же логике. Например для В3 это `enp0s8.113`, `enp0s9.213`, `enp0s8.813`.

### Адресация ISP

| Сеть | В1 | В2 | В3 | В4 |
|---|---|---|---|---|
| ISP → HQ-RTR | 172.16.10.0/28 | 172.16.30.0/28 | 172.16.50.0/28 | 172.16.70.0/28 |
| ISP → BR-RTR | 172.16.20.0/28 | 172.16.40.0/28 | 172.16.60.0/28 | 172.16.80.0/28 |

Из сети /28 первый адрес (x.x.x.1) идёт ISP, второй (x.x.x.2) идёт роутеру офиса. Шлюз HQ-RTR и BR-RTR это .1 своей сети.

Пример для В1:
- ISP enp0s8: 172.16.10.1/28, ISP enp0s9: 172.16.20.1/28
- HQ-RTR enp0s3: 172.16.10.2/28, шлюз 172.16.10.1
- BR-RTR enp0s3: 172.16.20.2/28, шлюз 172.16.20.1

DNS-записи docker и web указывают на интерфейсы ISP:
- `docker.au-team.irpo` → адрес ISP в сторону HQ-RTR
- `web.au-team.irpo` → адрес ISP в сторону BR-RTR

### Пользователи и SSH

| Параметр | В1 | В2 | В3 | В4 |
|---|---|---|---|---|
| Имя SSH-пользователя | sshuser | remote_user | sshuser | sshuser |
| UID | 2011 | 2012 | 2013 | 2014 |
| Пароль | P@ssw0rd | P@ssw0rd | P@ssw0rd | P@ssw0rd |
| SSH порт | 2011 | 2012 | 2013 | 2014 |

В варианте В2 имя пользователя `remote_user`, а не `sshuser`. В sshd_config на HQ-SRV и BR-SRV для В2 пиши `AllowUsers remote_user`.

Команда создания (подставь нужный UID и имя):

```bash
# В1
useradd -m -u 2011 -s /bin/bash sshuser
echo 'sshuser:P@ssw0rd' | chpasswd
echo 'sshuser ALL=(ALL) NOPASSWD: ALL' > /etc/sudoers.d/sshuser
chmod 440 /etc/sudoers.d/sshuser

# В2 (имя remote_user, UID 2012)
useradd -m -u 2012 -s /bin/bash remote_user
echo 'remote_user:P@ssw0rd' | chpasswd
echo 'remote_user ALL=(ALL) NOPASSWD: ALL' > /etc/sudoers.d/remote_user
chmod 440 /etc/sudoers.d/remote_user

# В3 (UID 2013)
useradd -m -u 2013 -s /bin/bash sshuser
echo 'sshuser:P@ssw0rd' | chpasswd
echo 'sshuser ALL=(ALL) NOPASSWD: ALL' > /etc/sudoers.d/sshuser
chmod 440 /etc/sudoers.d/sshuser

# В4 (UID 2014)
useradd -m -u 2014 -s /bin/bash sshuser
echo 'sshuser:P@ssw0rd' | chpasswd
echo 'sshuser ALL=(ALL) NOPASSWD: ALL' > /etc/sudoers.d/sshuser
chmod 440 /etc/sudoers.d/sshuser
```

SSH-конфиг `/etc/openssh/sshd_config` (подставь нужный порт и имя юзера):

```
Port <2011|2012|2013|2014>
AllowUsers <sshuser|remote_user>
MaxAuthTries 2
Banner /etc/openssh/banner
```

## Модуль 2. Различия

### RAID

| Параметр | В1 | В2 | В3 | В4 |
|---|---|---|---|---|
| Уровень | RAID 1 | RAID 5 | RAID 5 | RAID 5 |
| Дисков | 2 | 3 | 3 | 3 |
| Имя устройства | md1 | md2 | md3 | md4 |
| Точка монтирования | /raid | /raid | /raid | /raid |

Команды создания RAID (замени mdN и уровень):

```bash
# В1 — RAID 1, два диска, имя md1
mdadm --create /dev/md1 --level=1 --raid-devices=2 /dev/sdb /dev/sdc
mkfs.ext4 /dev/md1
mkdir /raid
mount /dev/md1 /raid
mdadm --detail --scan >> /etc/mdadm.conf
echo '/dev/md1  /raid  ext4  defaults  0 0' >> /etc/fstab

# В2 — RAID 5, три диска, имя md2
mdadm --create /dev/md2 --level=5 --raid-devices=3 /dev/sdb /dev/sdc /dev/sdd
mkfs.ext4 /dev/md2
mkdir /raid
mount /dev/md2 /raid
mdadm --detail --scan >> /etc/mdadm.conf
echo '/dev/md2  /raid  ext4  defaults  0 0' >> /etc/fstab

# В3 — RAID 5, три диска, имя md3
mdadm --create /dev/md3 --level=5 --raid-devices=3 /dev/sdb /dev/sdc /dev/sdd
mkfs.ext4 /dev/md3
mkdir /raid
mount /dev/md3 /raid
mdadm --detail --scan >> /etc/mdadm.conf
echo '/dev/md3  /raid  ext4  defaults  0 0' >> /etc/fstab

# В4 — RAID 5, три диска, имя md4
mdadm --create /dev/md4 --level=5 --raid-devices=3 /dev/sdb /dev/sdc /dev/sdd
mkfs.ext4 /dev/md4
mkdir /raid
mount /dev/md4 /raid
mdadm --detail --scan >> /etc/mdadm.conf
echo '/dev/md4  /raid  ext4  defaults  0 0' >> /etc/fstab
```

После создания RAID папка NFS и экспорт одинаковые для всех вариантов:

```bash
mkdir -p /raid/nfs
# /etc/exports:
/raid/nfs <сеть_HQ-CLI>/28(rw,no_root_squash,subtree_check)
exportfs -ra
```

### Chrony stratum

| В1 | В2 | В3 | В4 |
|---|---|---|---|
| 6 | 7 | 8 | 9 |

Chrony в этих вариантах настраивается на ISP, а не на HQ-RTR. Клиенты: HQ-SRV, HQ-CLI, BR-RTR, BR-SRV.

ISP `/etc/chrony.conf`:

```
pool pool.ntp.org iburst
local stratum <6|7|8|9>
allow 0.0.0.0/0
```

Клиенты, заменить pool на:

```
server <IP_ISP_в_сторону_HQ> iburst
```

```bash
systemctl enable --now chronyd
chronyc tracking
```

### Docker приложение

| Параметр | В1 | В2 | В3 | В4 |
|---|---|---|---|---|
| Образ БД | mariadb_latest | mariadb_latest | postgresql_latest | postgresql_latest |
| Имя БД | testdb1 | testdb2 | testdb3 | testdb4 |
| Пользователь БД | test1 | test2 | test3 | test4 |
| Пароль | P@ssw0rd | P@ssw0rd | P@ssw0rd | P@ssw0rd |
| Порт приложения | 8081 | 8082 | 8083 | 8084 |
| Контейнер site | site | site | site | site |
| Контейнер БД | db | db | db | db |

Образы загружены в `/docker` образа `Additional.iso`. Сначала импортируй:

```bash
docker load -i /mnt/docker/site_latest.tar
docker load -i /mnt/docker/mariadb_latest.tar    # В1, В2
docker load -i /mnt/docker/postgresql_latest.tar # В3, В4
```

Пример yaml для В1 (`~/app.yml`):

```yaml
version: '3'
services:
  site:
    image: site_latest
    container_name: site
    ports:
      - '8081:8081'
    environment:
      DB_HOST: db
      DB_NAME: testdb1
      DB_USER: test1
      DB_PASSWORD: P@ssw0rd
    depends_on:
      - db
  db:
    image: mariadb_latest
    container_name: db
    environment:
      MYSQL_DATABASE: testdb1
      MYSQL_USER: test1
      MYSQL_PASSWORD: P@ssw0rd
      MYSQL_ROOT_PASSWORD: root
```

Для В3/В4 с postgresql замени переменные окружения БД:

```yaml
  db:
    image: postgresql_latest
    container_name: db
    environment:
      POSTGRES_DB: testdb3
      POSTGRES_USER: test3
      POSTGRES_PASSWORD: P@ssw0rd
```

```bash
docker compose -f ~/app.yml up -d
```

### Проброс портов

| Маршрутизатор | В1 | В2 | В3 | В4 |
|---|---|---|---|---|
| HQ-RTR: веб → HQ-SRV | 8081 | 8082 | 8083 | 8084 |
| BR-RTR: docker → BR-SRV | 8081 | 8082 | 8083 | 8084 |
| HQ-RTR: SSH → HQ-SRV | 2011 | 2012 | 2013 | 2014 |
| BR-RTR: SSH → BR-SRV | 2011 | **2026** | 2013 | 2014 |

Обрати внимание: в В2 SSH на BR-RTR пробрасывается на порт **2026**, а не 2012.

```bash
# HQ-RTR (подставь порты варианта)
WEBPORT=8081
SSHPORT=2011
iptables -t nat -A PREROUTING -i enp0s3 -p tcp --dport $WEBPORT -j DNAT --to-destination 192.168.10.2:$WEBPORT
iptables -t nat -A PREROUTING -i enp0s3 -p tcp --dport $SSHPORT -j DNAT --to-destination 192.168.10.2:$SSHPORT
iptables -A FORWARD -p tcp -d 192.168.10.2 --dport $WEBPORT -j ACCEPT
iptables -A FORWARD -p tcp -d 192.168.10.2 --dport $SSHPORT -j ACCEPT
iptables-save > /etc/sysconfig/iptables

# BR-RTR (В2: SSHPORT_BR=2026)
WEBPORT=8081
SSHPORT_BR=2011
iptables -t nat -A PREROUTING -i enp0s3 -p tcp --dport $WEBPORT -j DNAT --to-destination 192.168.30.2:$WEBPORT
iptables -t nat -A PREROUTING -i enp0s3 -p tcp --dport $SSHPORT_BR -j DNAT --to-destination 192.168.30.2:$SSHPORT_BR
iptables -A FORWARD -p tcp -d 192.168.30.2 --dport $WEBPORT -j ACCEPT
iptables -A FORWARD -p tcp -d 192.168.30.2 --dport $SSHPORT_BR -j ACCEPT
iptables-save > /etc/sysconfig/iptables
```

### nginx basic auth

| В1 | В2 | В3 | В4 |
|---|---|---|---|
| Khariton | Kozma | Kazimir | Erofey |

Пароль везде `P@ssw0rd`. Настройка на ISP:

```bash
apt-get install nginx apache2-utils -y
htpasswd -cb /etc/nginx/.htpasswd Khariton P@ssw0rd
```

`/etc/nginx/conf.d/proxy.conf`:

```nginx
server {
    listen 80;
    server_name web.au-team.irpo;
    auth_basic "Restricted";
    auth_basic_user_file /etc/nginx/.htpasswd;
    location / {
        proxy_pass http://<IP_HQ-RTR>:8081;
        proxy_set_header Host $host;
    }
}
server {
    listen 80;
    server_name docker.au-team.irpo;
    location / {
        proxy_pass http://<IP_BR-RTR>:8081;
        proxy_set_header Host $host;
    }
}
```

Аутентификация только на web.au-team.irpo, для docker без неё.

```bash
nginx -t && systemctl enable --now nginx
```

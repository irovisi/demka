## 2. Базовая настройка устройств

### Имена (FQDN)

На каждой машине свою команду, затем перелогинься через `exec bash`.

```bash
hostnamectl set-hostname hq-rtr.au-team.irpo   # на HQ-RTR
hostnamectl set-hostname br-rtr.au-team.irpo   # на BR-RTR
hostnamectl set-hostname hq-srv.au-team.irpo   # на HQ-SRV
hostnamectl set-hostname hq-cli.au-team.irpo   # на HQ-CLI
hostnamectl set-hostname br-srv.au-team.irpo   # на BR-SRV
```

### IPv4 на простых интерфейсах

Адреса HQ-SRV, HQ-CLI и офисных интерфейсов HQ-RTR настраиваются в разделе 5 (VLAN). Здесь только прямые L3-интерфейсы.

HQ-RTR в сторону ISP:

```bash
cat > /etc/net/ifaces/enp0s3/options <<'EOF'
BOOTPROTO=static
CONFIG_IPV4=yes
ONBOOT=yes
TYPE=eth
DISABLED=no
EOF
echo "172.16.4.2/28" > /etc/net/ifaces/enp0s3/ipv4address
echo "default via 172.16.4.1" > /etc/net/ifaces/enp0s3/ipv4route
systemctl restart network
```

BR-RTR в сторону ISP и в сторону BR-SRV:

```bash
cat > /etc/net/ifaces/enp0s3/options <<'EOF'
BOOTPROTO=static
CONFIG_IPV4=yes
ONBOOT=yes
TYPE=eth
DISABLED=no
EOF
echo "172.16.5.2/28" > /etc/net/ifaces/enp0s3/ipv4address
echo "default via 172.16.5.1" > /etc/net/ifaces/enp0s3/ipv4route

cat > /etc/net/ifaces/enp0s8/options <<'EOF'
BOOTPROTO=static
CONFIG_IPV4=yes
ONBOOT=yes
TYPE=eth
DISABLED=no
EOF
echo "192.168.30.1/27" > /etc/net/ifaces/enp0s8/ipv4address
systemctl restart network
```

BR-SRV:

```bash
cat > /etc/net/ifaces/enp0s3/options <<'EOF'
BOOTPROTO=static
CONFIG_IPV4=yes
ONBOOT=yes
```

## 3. Настройка ISP

ISP это маршрутизатор провайдера. Магистраль по DHCP, два интерфейса вниз статикой, NAT для офисов.

```bash
# Магистраль (интернет) по DHCP
cat > /etc/net/ifaces/enp0s3/options <<'EOF'
BOOTPROTO=dhcp
CONFIG_IPV4=yes
ONBOOT=yes
TYPE=eth
DISABLED=no
EOF

# В сторону HQ-RTR
cat > /etc/net/ifaces/enp0s8/options <<'EOF'
BOOTPROTO=static
CONFIG_IPV4=yes
ONBOOT=yes
TYPE=eth
DISABLED=no
EOF
echo "172.16.4.1/28" > /etc/net/ifaces/enp0s8/ipv4address

# В сторону BR-RTR
cat > /etc/net/ifaces/enp0s9/options <<'EOF'
BOOTPROTO=static
CONFIG_IPV4=yes
ONBOOT=yes
TYPE=eth
DISABLED=no
EOF
echo "172.16.5.1/28" > /etc/net/ifaces/enp0s9/ipv4address
systemctl restart network
```

Маршрут по умолчанию приходит от провайдера через DHCP. Включи форвардинг:

```bash
sed -i 's/^net.ipv4.ip_forward.*/net.ipv4.ip_forward = 1/' /etc/net/sysctl.conf
sysctl -w net.ipv4.ip_forward=1
```

Динамическая трансляция в сторону интернета для сетей HQ-RTR и BR-RTR:

```bash
iptables -t nat -A POSTROUTING -s 172.16.4.0/28 -o enp0s3 -j MASQUERADE
iptables -t nat -A POSTROUTING -s 172.16.5.0/28 -o enp0s3 -j MASQUERADE
iptables -A FORWARD -i enp0s8 -o enp0s3 -j ACCEPT
iptables -A FORWARD -i enp0s9 -o enp0s3 -j ACCEPT
iptables -A FORWARD -m state --state ESTABLISHED,RELATED -j ACCEPT
iptables-save > /etc/sysconfig/iptables
systemctl enable iptables
systemctl start iptables
```

Офисные сети 192.168.x транслируются ещё на HQ-RTR и BR-RTR (раздел 9), поэтому до ISP они доходят уже с адресами 172.16.4.x и 172.16.5.x.

## 4. Локальные учётные записи

### sshuser на HQ-SRV и BR-SRV

UID 1010, пароль `P@ssw0rd`, sudo без пароля.

```bash
useradd -m -u 1010 -s /bin/bash sshuser
echo 'sshuser:P@ssw0rd' | chpasswd
echo 'sshuser ALL=(ALL) NOPASSWD: ALL' > /etc/sudoers.d/sshuser
chmod 440 /etc/sudoers.d/sshuser
```

### net_admin на HQ-RTR и BR-RTR

Пароль `P@$$word`, sudo без пароля. Одинарные кавычки сохраняют `$$` в пароле.

```bash
useradd -m -s /bin/bash net_admin
echo 'net_admin:P@$$word' | chpasswd
echo 'net_admin ALL=(ALL) NOPASSWD: ALL' > /etc/sudoers.d/net_admin
chmod 440 /etc/sudoers.d/net_admin
```

Если HQ-RTR / BR-RTR на EcoRouter, а не на Linux: создай net_admin с паролем `P@$$word` и максимальными привилегиями (`privilege 15`) средствами CLI EcoRouter.

## 5. Виртуальный коммутатор и VLAN на HQ-RTR

Каналы прямые, отдельного коммутатора нет. Реализация: 802.1Q-тегирование на каждом канале. enp0s8 (к HQ-SRV) несёт VLAN 100 и VLAN 999, enp0s9 (к HQ-CLI) несёт VLAN 200. Физические интерфейсы работают как trunk без IP, адреса вынесены на VLAN-сабинтерфейсы.

> Выполняй с консоли, не по SSH через перенастраиваемый интерфейс, иначе оборвёшь сессию.

### HQ-RTR

```bash
modprobe 8021q
echo 8021q > /etc/modules-load.d/8021q.conf

# enp0s8 trunk без IP
rm -f /etc/net/ifaces/enp0s8/ipv4address
cat > /etc/net/ifaces/enp0s8/options <<'EOF'
BOOTPROTO=static
CONFIG_IPV4=no
ONBOOT=yes
TYPE=eth
DISABLED=no
EOF

# VLAN 100 (HQ-SRV)
mkdir -p /etc/net/ifaces/enp0s8.100
cat > /etc/net/ifaces/enp0s8.100/options <<'EOF'
BOOTPROTO=static
CONFIG_IPV4=yes
HOST=enp0s8
ONBOOT=yes
TYPE=vlan
VID=100
DISABLED=no
EOF
echo "192.168.10.1/26" > /etc/net/ifaces/enp0s8.100/ipv4address

# VLAN 999 (управление)
mkdir -p /etc/net/ifaces/enp0s8.999
cat > /etc/net/ifaces/enp0s8.999/options <<'EOF'
BOOTPROTO=static
CONFIG_IPV4=yes
HOST=enp0s8
ONBOOT=yes
TYPE=vlan
VID=999
DISABLED=no
EOF
echo "192.168.99.1/29" > /etc/net/ifaces/enp0s8.999/ipv4address

# enp0s9 trunk без IP
rm -f /etc/net/ifaces/enp0s9/ipv4address
cat > /etc/net/ifaces/enp0s9/options <<'EOF'
BOOTPROTO=static
CONFIG_IPV4=no
ONBOOT=yes
TYPE=eth
DISABLED=no
EOF

# VLAN 200 (HQ-CLI)
mkdir -p /etc/net/ifaces/enp0s9.200
cat > /etc/net/ifaces/enp0s9.200/options <<'EOF'
BOOTPROTO=static
CONFIG_IPV4=yes
HOST=enp0s9
ONBOOT=yes
TYPE=vlan
VID=200
DISABLED=no
EOF
echo "192.168.20.1/28" > /etc/net/ifaces/enp0s9.200/ipv4address

sed -i 's/^net.ipv4.ip_forward.*/net.ipv4.ip_forward = 1/' /etc/net/sysctl.conf
systemctl restart network
```

### HQ-SRV (тег VLAN 100)

```bash

echo 8021q > /etc/modules-load.d/8021q.conf

rm -f /etc/net/ifaces/enp0s3/ipv4address /etc/net/ifaces/enp0s3/ipv4route

ONBOOT=yes
TYPE=eth
DISABLED=no

mkdir -p /etc/net/ifaces/enp0s3.100
cat > /etc/net/ifaces/enp0s3.100/options <<'EOF'

CONFIG_IPV4=yes
HOST=enp0s3
ONBOOT=yes

VID=100

EOF
echo "192.168.10.2/26" > /etc/net/ifaces/enp0s3.100/ipv4address
echo "default via 192.168.10.1" > /etc/net/ifaces/enp0s3.100/ipv4route

```

### HQ-CLI (тег VLAN 200, временно статика)

Позже станет DHCP-клиентом в разделе 10. Сейчас статика нужна для проверки VLAN.

```bash

echo 8021q > /etc/modules-load.d/8021q.conf

rm -f /etc/net/ifaces/enp0s3/ipv4address /etc/net/ifaces/enp0s3/ipv4route

EOF

mkdir -p /etc/net/ifaces/enp0s3.200
cat > /etc/net/ifaces/enp0s3.200/options <<'EOF'
BOOTPROTO=static
CONFIG_IPV4=yes
HOST=enp0s3
ONBOOT=yes
TYPE=vlan
VID=200
DISABLED=no
EOF
echo "192.168.20.2/28" > /etc/net/ifaces/enp0s3.200/ipv4address
echo "default via 192.168.20.1" > /etc/net/ifaces/enp0s3.200/ipv4route
```

### Проверка

```bash
ip -br a            # на HQ-RTR видны enp0s8.100, enp0s8.999, enp0s9.200
# с HQ-CLI
ping 192.168.20.1   # шлюз
ping 192.168.10.2   # HQ-SRV через межвлановую маршрутизацию
```

### В отчёт

Разделение на VLAN выполнено 802.1Q-тегированием на каждом прямом канале средствами etcnet. Физические интерфейсы HQ-RTR работают как trunk без IP, адреса вынесены на сабинтерфейсы. VLAN 100 (HQ-SRV): 192.168.10.0/26, шлюз 192.168.10.1, канал enp0s8. VLAN 200 (HQ-CLI): 192.168.20.0/28, шлюз 192.168.20.1, канал enp0s9. VLAN 999 (управление): 192.168.99.0/29, шлюз 192.168.99.1, транк на канале enp0s8. Межвлановая маршрутизация включена через net.ipv4.ip_forward=1. Альтернатива по заданию это один транк к коммутатору со всеми тремя VLAN, но в стенде два прямых линка, поэтому реализация поканальная.

## 6. Безопасный SSH на HQ-SRV и BR-SRV

Порт 2024, доступ только sshuser, две попытки входа, баннер.

```bash
sed -i 's/^#\?Port .*/Port 2024/' /etc/openssh/sshd_config
sed -i 's/^#\?MaxAuthTries .*/MaxAuthTries 2/' /etc/openssh/sshd_config

# Дописать, если строк нет
grep -q '^AllowUsers' /etc/openssh/sshd_config || echo 'AllowUsers sshuser' >> /etc/openssh/sshd_config
grep -q '^Banner'    /etc/openssh/sshd_config || echo 'Banner /etc/openssh/banner' >> /etc/openssh/sshd_config

echo 'Authorized access only' > /etc/openssh/banner
systemctl restart sshd
```

Проверка с другой машины: `ssh -p 2024 sshuser@<адрес>` показывает баннер и пускает только sshuser.

## 7. GRE-туннель между HQ и BR

Технология GRE, сеть туннеля 10.10.10.0/30. Эндпоинты это внешние адреса роутеров, ISP маршрутизирует между 172.16.4.0/28 и 172.16.5.0/28, так как подключён к обоим напрямую.

### HQ-RTR (etcnet, с автозагрузкой)

```bash
mkdir -p /etc/net/ifaces/gre1
cat > /etc/net/ifaces/gre1/options <<'EOF'
TYPE=iptun
TUNTYPE=gre
TUNLOCAL=172.16.4.2
TUNREMOTE=172.16.5.2
TUNOPTIONS=ttl 255
HOST=enp0s3
ONBOOT=yes
CONFIG_IPV4=yes
DISABLED=no
EOF
echo "10.10.10.1/30" > /etc/net/ifaces/gre1/ipv4address
systemctl restart network
```

### BR-RTR

```bash
mkdir -p /etc/net/ifaces/gre1
cat > /etc/net/ifaces/gre1/options <<'EOF'
TYPE=iptun
TUNTYPE=gre
TUNLOCAL=172.16.5.2
TUNREMOTE=172.16.4.2
TUNOPTIONS=ttl 255
HOST=enp0s3
ONBOOT=yes
CONFIG_IPV4=yes
DISABLED=no
EOF
echo "10.10.10.2/30" > /etc/net/ifaces/gre1/ipv4address
systemctl restart network
```

Быстрая альтернатива без автозагрузки (если etcnet не поднимает gre):

```bash
# HQ-RTR
ip tunnel add gre1 mode gre local 172.16.4.2 remote 172.16.5.2 ttl 255
ip addr add 10.10.10.1/30 dev gre1 && ip link set gre1 up
# BR-RTR
ip tunnel add gre1 mode gre local 172.16.5.2 remote 172.16.4.2 ttl 255
ip addr add 10.10.10.2/30 dev gre1 && ip link set gre1 up
```

Проверка: `ping 10.10.10.2` с HQ-RTR.

### В отчёт

Туннель GRE между HQ-RTR и BR-RTR. Локальные/удалённые адреса 172.16.4.2 ↔ 172.16.5.2. Транспортная сеть туннеля 10.10.10.0/30: HQ-RTR 10.10.10.1, BR-RTR 10.10.10.2.

## 8. Динамическая маршрутизация OSPF

Link-state протокол это OSPF, на ALT поднимается через FRR. Протокол активен только на gre1, соседство только между двумя роутерами, защита паролем. Подробный траблшутинг в [[OSPF]].

### Подготовка (оба роутера)

```bash
apt-get install frr -y
sed -i 's/^ospfd=no/ospfd=yes/' /etc/frr/daemons
systemctl enable --now frr
```

### HQ-RTR

```bash
vtysh
```

```
configure terminal
router ospf
 ospf router-id 1.1.1.1
 passive-interface default
 network 10.10.10.0/30 area 0
 network 192.168.10.0/26 area 0
 network 192.168.20.0/28 area 0
 network 192.168.99.0/29 area 0
exit
interface gre1
 no ip ospf passive
 ip ospf authentication
 ip ospf authentication-key P@ssw0rd
exit
do write
exit
```

### BR-RTR

```
configure terminal
router ospf
 ospf router-id 2.2.2.2
 passive-interface default
 network 10.10.10.0/30 area 0
 network 192.168.30.0/27 area 0
exit
interface gre1
 no ip ospf passive
 ip ospf authentication
 ip ospf authentication-key P@ssw0rd
exit
do write
exit
```

Как это закрывает требования: `passive-interface default` глушит OSPF на всех интерфейсах, `no ip ospf passive` на gre1 включает обратно только туннель, поэтому hello летят лишь между роутерами. Сети офисов объявлены через `network`, но их интерфейсы остаются passive, маршруты распространяются, чужие соседства не появляются. Парольная защита через `ip ospf authentication` + `authentication-key`.

### Проверка

```bash
vtysh -c "show ip ospf neighbor"   # сосед в состоянии Full
vtysh -c "show ip route ospf"      # сети другого офиса
```

Если сосед не поднимается поверх GRE, частая причина это rp_filter. Сбрось его и проверь совпадение ключа (см. [[OSPF]]):

```bash
sysctl -w net.ipv4.conf.all.rp_filter=0
sysctl -w net.ipv4.conf.gre1.rp_filter=0
echo 'net.ipv4.conf.all.rp_filter=0' >> /etc/net/sysctl.conf
```

### В отчёт

Протокол OSPF, area 0. Активен только на gre1 за счёт `passive-interface default` и `no ip ospf passive` на туннеле. Сети офисов распространяются через `network`. Защита простой парольной аутентификацией, ключ `P@ssw0rd`.

## 9. NAT на маршрутизаторах

Каждый офис транслирует свои адреса в сторону ISP. Трансляция привязана к интерфейсу enp0s3, поэтому трафик между офисами через gre1 не транслируется и ходит по маршрутам OSPF.

Сначала форвардинг на обоих роутерах:

```bash
sed -i 's/^net.ipv4.ip_forward.*/net.ipv4.ip_forward = 1/' /etc/net/sysctl.conf
sysctl -w net.ipv4.ip_forward=1
```

### HQ-RTR

```bash
iptables -t nat -A POSTROUTING -s 192.168.10.0/26 -o enp0s3 -j MASQUERADE
iptables -t nat -A POSTROUTING -s 192.168.20.0/28 -o enp0s3 -j MASQUERADE
iptables -t nat -A POSTROUTING -s 192.168.99.0/29 -o enp0s3 -j MASQUERADE
iptables-save > /etc/sysconfig/iptables
systemctl enable iptables && systemctl start iptables
```

### BR-RTR

```bash
iptables -t nat -A POSTROUTING -s 192.168.30.0/27 -o enp0s3 -j MASQUERADE
iptables-save > /etc/sysconfig/iptables
systemctl enable iptables && systemctl start iptables
```

Проверка: с HQ-CLI `ping 8.8.8.8` проходит через двойной NAT (HQ-RTR, затем ISP).

## 10. DHCP на HQ-RTR

Сервер DHCP это HQ-RTR, клиент HQ-CLI в VLAN 200. Адрес роутера исключён, шлюз и DNS заданы, суффикс au-team.irpo.

```bash
apt-get install dhcp-server -y
```

`/etc/dhcp/dhcpd.conf`:

```
option domain-name "au-team.irpo";
default-lease-time 600;
max-lease-time 7200;

subnet 192.168.20.0 netmask 255.255.255.240 {
    range 192.168.20.2 192.168.20.14;
    option routers 192.168.20.1;
    option domain-name-servers 192.168.10.2;
    option domain-name "au-team.irpo";
}
```

Маска /28 это 255.255.255.240. Диапазон начат с .2, поэтому адрес роутера .1 в выдачу не попадает. DNS указывает на HQ-SRV.

Привяжи сервис к интерфейсу VLAN 200 и запусти:

```bash
echo 'DHCPDARGS=enp0s9.200' > /etc/sysconfig/dhcpd
systemctl enable --now dhcpd
```

Переведи HQ-CLI на DHCP:

```bash
rm -f /etc/net/ifaces/enp0s3.200/ipv4address /etc/net/ifaces/enp0s3.200/ipv4route
sed -i 's/^BOOTPROTO=static/BOOTPROTO=dhcp/' /etc/net/ifaces/enp0s3.200/options
systemctl restart network
ip -br a   # должен прийти 192.168.20.2
```

### В отчёт

DHCP на HQ-RTR обслуживает VLAN 200 (192.168.20.0/28). Пул .2–.14, шлюз 192.168.20.1, DNS 192.168.10.2, суффикс au-team.irpo. Адрес роутера .1 исключён началом диапазона.

## 11. DNS на HQ-SRV

Основной сервер на HQ-SRV (BIND). Прямая и обратная зоны по таблице 2, пересылка на публичный DNS.

```bash
apt-get install bind bind-utils -y
```

`/etc/bind/options.conf`:

```
options {
    directory "/etc/bind/zone";
    dnssec-validation no;
    listen-on { any; };
    forwarders { 8.8.8.8; };
    allow-query { any; };
    allow-recursion { any; };
};
zone "au-team.irpo" {
    type master;
    file "/etc/bind/zone/au-team.irpo";
};
zone "168.192.in-addr.arpa" {
    type master;
    file "/etc/bind/zone/168.192.in-addr.arpa";
};
```

`/etc/bind/zone/au-team.irpo`:

```
$TTL 1D
@   IN SOA hq-srv.au-team.irpo. root.au-team.irpo. (
        2026061700 ; serial
        12H 1H 1W 1H )
@        IN NS    hq-srv.au-team.irpo.
hq-rtr   IN A     192.168.10.1
br-rtr   IN A     192.168.30.1
hq-srv   IN A     192.168.10.2
hq-cli   IN A     192.168.20.2
br-srv   IN A     192.168.30.2
moodle   IN A     172.16.4.1
wiki     IN A     172.16.5.1
```

moodle и wiki это A-записи на интерфейсы ISP по таблице 2: moodle на интерфейс в сторону HQ-RTR (172.16.4.1), wiki на интерфейс в сторону BR-RTR (172.16.5.1). На этих адресах в модуле 2 поднимается обратный прокси nginx.

`/etc/bind/zone/168.192.in-addr.arpa` (PTR только для hq-rtr, hq-srv, hq-cli):

```
$TTL 1D
@   IN SOA hq-srv.au-team.irpo. root.au-team.irpo. (
        2026061700 ; serial
        12H 1H 1W 1H )
@      IN NS  hq-srv.au-team.irpo.
1.10   IN PTR hq-rtr.au-team.irpo.
2.10   IN PTR hq-srv.au-team.irpo.
2.20   IN PTR hq-cli.au-team.irpo.
```

Запись `2.20` это адрес 192.168.20.2 в зоне 168.192. HQ-CLI должен получить .2 от DHCP (начало пула), тогда PTR совпадёт.

Проверь и запусти:

```bash
chown root:named /etc/bind/zone/au-team.irpo /etc/bind/zone/168.192.in-addr.arpa
named-checkconf
named-checkzone au-team.irpo /etc/bind/zone/au-team.irpo
named-checkzone 168.192.in-addr.arpa /etc/bind/zone/168.192.in-addr.arpa
systemctl enable --now bind
```

Сам HQ-SRV должен смотреть на себя: `echo -e 'search au-team.irpo\nnameserver 127.0.0.1' > /etc/resolv.conf`. HQ-CLI получает DNS через DHCP. Остальные серверы при необходимости укажи на 192.168.10.2.

Проверка:

```bash
nslookup hq-srv.au-team.irpo 192.168.10.2
nslookup 192.168.10.1 192.168.10.2
nslookup ya.ru 192.168.10.2     # пересылка
```

### В отчёт

Основной DNS на HQ-SRV (BIND), зона au-team.irpo и обратная 168.192.in-addr.arpa. Прямые записи всех устройств, PTR для hq-rtr, hq-srv, hq-cli. moodle и wiki указывают на интерфейсы ISP. Пересылка на 8.8.8.8.
